## 2025-02-20 - Unrolling 3x3 Convolution Kernels (Sobel) in Java
**Learning:** In Java, flattening small kernel loops like 3x3 Sobel filters directly into unrolled arithmetic yields a dramatic speedup (often 2.5x to 4x). The overhead from a nested 3x3 loop, including bounds-checking and arithmetic for computing array indexes (e.g. `(r+kr)*pW+(c+kc)` vs pre-calculated row offsets `row0 + c`), is surprisingly large relative to the actual math being performed.
**Action:** When identifying mathematical kernels like Sobel filters in image or heightmap processing loops, unroll the 3x3 array accesses directly, use local row offset variables, and multiply by the inverse (e.g., `* 0.125f`) instead of dividing.
## 2025-02-23 - Avoiding 2D Array Lookups and Math.floor in Hot Loops
**Learning:** In Java, 2D arrays are arrays-of-arrays. In hot mathematical loops (like bilinear resizing or gaussian blur), doing `array[r][c]` inside the inner column loop requires the JVM to perform bounds checking and pointer dereferencing for `array[r]` repeatedly. Hoisting `float[] row = array[r]` outside the inner loop provides a measurable speedup. Additionally, `Math.floor` is relatively slow compared to manual integer casting with conditional negative correction. Lastly, pre-calculating horizontal interpolation weights outside the row loop saves thousands of redundant floating-point operations.
**Action:** When working with 2D arrays in tight image processing loops, always cache the 1D row array before entering the column loop. Use manual casts for `floor` when performance is critical. Pre-compute constants that don't change between rows.
